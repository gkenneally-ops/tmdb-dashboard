TMDB Movie Explorer — Project Report

An interactive Shiny dashboard exploring the economics of film

Author

Gerard Kenneally

Published

May 14, 2026

1 Introduction

The film industry has always run on a mix of confident projections and deeply uncertain outcomes. Studios spend hundreds of millions of dollars on individual films, marketed with precision and tracked obsessively, but the actual relationship between what goes into a film and what comes out of it is far messier than the trade press makes it sound. A $200 million superhero movie can flop. A $5 million horror film can earn $200 million. The averages get reported; the variance rarely does.

This project is a small attempt to make that variance visible. The dashboard is built in Shiny, deployed at https://gkenneally.shinyapps.io/tmdb-dashboard/, with source code at https://github.com/gkenneally-ops/tmdb-dashboard. It lets a user filter roughly 4,800 films from The Movie Database by year, genre, and popularity threshold, and explore six different angles on what those films looked like — financially, critically, and in terms of who made them.

The goal isn’t to predict box office numbers or rank “the best” films. It’s to give someone curious about the film industry a tool to ask their own questions and see the answers in the underlying data, instead of taking a single static chart on faith. The report below walks through the dataset, the questions the dashboard was built to answer, what those questions actually surfaced once the data was filtered and plotted, the design choices made along the way, and what a future version would do differently.

2 The data

2.1 Source and structure

The dashboard is built on the TMDB 5000 Movie Dataset, a publicly available collection of metadata distributed on Kaggle and originally sourced from The Movie Database (TMDB), an open community-maintained film and television catalog. The data can be downloaded at https://www.kaggle.com/datasets/tmdb/tmdb-movie-metadata.

The dataset comes in two CSV files. The first, tmdb_5000_movies.csv, contains roughly 4,800 films with around 20 attributes each: title, release date, budget, revenue, runtime, average user rating, vote count, popularity score, and several JSON-encoded columns capturing genres, keywords, production companies, production countries, and spoken languages. The second, tmdb_5000_credits.csv, contains the same set of films keyed by movie ID, with cast and crew information stored as longer JSON objects. The two files are joined on id ↔︎ movie_id to produce a single working dataset.

After cleaning, the analysis dataset contains 4,803 released films spanning from the 1910s through 2017 — comfortably above the project’s minimum of 500 observations and 5 variables. The dataset is heavily skewed toward more recent decades; about two-thirds of the films are from 1990 onward, which reflects both the increasing volume of film production and TMDB’s coverage tilt toward more recent titles.

2.2 Cleaning and derived variables

Several preparation steps were necessary before the data could power a useful dashboard.

The most consequential issue is that TMDB stores unknown budget and revenue figures as 0, not as missing values. A substantial fraction of films in the dataset — roughly a quarter — have a reported budget or revenue of zero. These aren’t films that genuinely cost or earned nothing; they’re films for which TMDB’s contributors haven’t entered financial information. Treating those zeros as real values would corrupt every downstream analysis: average budgets would crash, ROI calculations would explode toward zero or infinity, and the headline numbers in the dashboard would be meaningless. The data preparation step in global.R therefore replaces these zeros with explicit NA values, and any analysis involving budget, revenue, profit, or ROI filters them out.

The JSON-encoded columns required parsing. A small helper function, parse_json_names(), extracts the name field from each JSON array and returns a character vector. This produces a list-column for genres, allowing a single film to retain all of its genre tags rather than being collapsed to a single primary one. A separate primary_genre column (the first listed genre) is also derived for cases where a single label is needed, like the Top Films table. Filtering by genre in the dashboard uses the full list-column, so a film tagged as both “Action” and “Adventure” appears under either filter.

A second helper, parse_directors(), walks the crew JSON column and pulls out only the entries where job == "Director". The crew column lists every credited person on each film — gaffers, set decorators, second unit directors, the works — and most of them aren’t useful for a leaderboard. Filtering on the job field cleans that up. The result is a long table with one row per film-director pair, which the Directors tab aggregates back up to the person level.

Several derived variables support the dashboard’s analyses: release_year and decade from release_date for time-based filtering, profit as revenue − budget, and roi as revenue / budget. These all live in global.R so they’re computed once at app startup rather than recalculated on every reactive update.

2.3 Ethical considerations

The dataset is about commercially released films and the public-figure cast and crew who made them. It doesn’t contain personally identifiable information about private individuals, so the privacy concerns that would apply to, say, a healthcare or social-media dataset don’t really apply here. That said, two ethical considerations are worth flagging because they shape how the dashboard’s findings should be read.

The first is coverage bias. TMDB is a community-maintained database, and its coverage skews heavily toward English-language and Western films. Any framing in the dashboard that uses superlatives — “highest-grossing,” “most profitable genre,” “top films” — is implicitly qualifying those superlatives to “among the films TMDB volunteers chose to catalog.” This is not the global film industry; it’s a particular slice of it, and the dashboard’s titles and About tab try to acknowledge this. A genuinely global film dataset would look different, and the patterns the dashboard surfaces shouldn’t be mistaken for universal truths about cinema.

The second is nominal versus real dollars. Budget and revenue figures in the dataset are nominal: a $10 million film from 1990 is recorded as $10 million, even though that figure represents substantially more purchasing power than $10 million in 2015. The dashboard doesn’t adjust for inflation, and reasonable people can disagree about whether it should. Adjusting to constant dollars makes long-term comparisons more honest but introduces its own complications: the choice of base year, the right price index for film production specifically, and the question of whether audiences themselves were spending in nominal or real terms. The dashboard takes the simpler path of presenting nominal dollars and flagging the limitation rather than imposing a particular adjustment. A reader interested specifically in long-run trends across many decades should treat the long tail of older films with appropriate caution.

3 Questions

Five questions emerged from the exploratory analysis as candidates for the dashboard. The current dashboard answers all five, distributed across six tabs (the About tab being the sixth, providing context rather than analysis).

The first question is foundational: how well does budget predict revenue, and how much spread is there in that relationship? The intuition that bigger-budget films make more money is widely held, but the strength of that relationship — and the size of the variance around it — is hard to feel without seeing the underlying data. A scatter plot with a break-even reference line answers this directly.

The second question shifts from individual films to genres: which genres deliver the highest typical return on investment? This is a different question from “which genre makes the most money in absolute terms.” Marvel-scale tentpoles bring in enormous revenue but also cost enormous amounts; horror films and animation, conversely, tend to be cheap to produce relative to what they earn. A median-ROI comparison cuts through the absolute-dollar noise to reveal where the per-dollar returns actually are.

The third question is about discoverability: which specific films stand out within whatever filter the user has applied? Someone exploring the data should be able to ask “which 1990s science fiction films had the highest revenue?” and get a concrete, ranked answer, with the financial details. This is the role of the Top Films table.

The fourth question is about reception: do critic-favored films and audience-popular films overlap, or diverge? The two measures are often discussed as if they’re proxies for each other — popular films are good, niche films aren’t — but a scatter plot should show how often that’s actually true. A single chart with rating on one axis and popularity on the other separates the films audiences and critics both love from the films one camp embraces and the other ignores.

The fifth question puts people on screen instead of films: which directors consistently appear in profitable films? The dataset already contains the credit information; the question is whether aggregating it to the director level surfaces patterns worth seeing. A sortable, filterable leaderboard answers it.

4 Findings

4.1 Budget is a strong but very noisy predictor of revenue

The Budget vs. Revenue tab plots every film with a known budget and revenue against a break-even reference line. The dynamic title computes the percentage of films in the current filter that earned more than they cost, and that number is striking depending on how the user has filtered. With the minimum-vote slider set high — say, 2,100 votes or more, which restricts the view to films that have actually been seen by a meaningful audience — the headline reads “99% of films in this view earned more than they cost.” With the slider at the default 50 votes, the figure is closer to 77%.

That gap is itself the finding. The 99% number isn’t because almost every film is profitable; it’s because the films visible enough to attract thousands of votes are heavily biased toward commercial success. The films that fail quietly, by definition, also tend to fail to accumulate ratings on TMDB. The dashboard’s filter exposes this selection effect rather than hiding it: dragging the minimum-vote slider from low to high doesn’t just change the chart density, it changes the answer to “are most films profitable?” That’s a more honest answer than any single number could give.

The visual pattern, regardless of filter, is one of strong correlation with enormous spread. The point cloud rises from lower-left to upper-right, confirming the basic intuition that bigger budgets are associated with bigger revenues. But the cloud is also extraordinarily wide. At any given budget level, the revenue range spans more than three orders of magnitude. A studio executive looking at the chart would be hard-pressed to argue that budget alone is a reliable predictor of outcomes. The dashed break-even line is genuinely close to the median film’s experience, and the variation around it dwarfs whatever signal is in the budget itself.

4.2 The “best ROI genre” depends entirely on how you filter

The Genre Comparison tab computes the median return on investment for each genre with at least ten films in the current filtered view, and highlights the top genre in red while muting the rest in gray. The interesting thing about this chart is that the answer changes substantially depending on the filter. With certain filter settings the leader is Romance; with others it’s Horror, Animation, or Family. Drag the year range earlier or set a higher minimum vote count and the ranking shuffles.

Horror’s strong median ROI when it ranks highly is the most well-documented pattern — the genre is cheap to produce, has a dedicated audience, and movies like the Saw franchise, Paranormal Activity, and The Blair Witch Project regularly return many multiples of their budgets. Animation benefits from a different structure: theatrical longevity, international markets, and merchandising tails that the data here doesn’t even capture. When Romance wins, it’s usually because films like Twilight or specific sleeper hits drag the median up despite a smaller film count.

What’s not at the top of the ROI ranking, in any filter setting, is what’s most interesting. Action and Adventure — the genres most associated with tentpole production and cultural dominance — sit closer to the middle. The reason is that their budgets are also enormous. A $200 million superhero film that earns $700 million is a 3.5× ROI, which is good but not exceptional in a relative sense. The dashboard’s choice to highlight only the top genre in the brand red, with everything else muted, makes the answer to “which genre wins?” pop preattentively while still letting the user verify the rest of the ranking on the same chart.

Drama, the largest genre by film count, sits in the middle of the pack regardless of filter. That makes sense: “drama” is a catch-all category that covers everything from $1 million festival films to $50 million prestige productions, and the resulting ROI distribution has so much internal variance that the median is unrevealing.

4.3 The Top Films table surfaces specific stories worth investigating

The Top Films tab presents a sortable table of the top 50 films by revenue within the current filter, with budget, revenue, ROI, and rating displayed for each. The table isn’t a finding in itself — it’s a tool for surfacing specific films that prompt further questions.

A few patterns recur. The top of the revenue ranking is dominated by recent franchise entries: Avatar, Titanic, The Avengers, Jurassic World, Furious 7, Avengers: Age of Ultron, Frozen, Iron Man 3, Minions, Captain America: Civil War. That’s the well-documented industry pattern of the 2010s playing out in the data — sequels, remakes, and shared-universe installments dominating the box office. ROI tells a different story: Minions stands out with a 15.6× return on a $74 million budget, well above the bigger-budget Marvel and DC entries that earn more in absolute terms but spend more to do it. The visual ROI bars inside each row (a horizontal red bar that fills proportionally) make these outliers easy to spot at a glance without having to read the numbers.

The table also exposes the limits of revenue as a quality measure. Films at the top of the revenue ranking aren’t necessarily at the top of the rating ranking, and several of the highest-rated films in the dataset never enter the Top 50 by revenue. That gap between commercial and critical success is exactly what the next tab addresses.

4.4 Critic-favored films and audience-popular films mostly don’t overlap

The Critics vs. Audience tab plots every filtered film with average rating on the y-axis and popularity score on the x-axis (log-scaled, since popularity is heavily right-skewed). Films are highlighted into three categories using percentile ranks within the current filter: Universally loved (top 10% on both axes, in red), Critic darlings (top 10% rating, bottom 50% popularity, in blue), and Crowd pleasers (top 10% popularity, bottom 50% rating, in yellow). The vast majority of films, in gray, sit in the middle on both dimensions.

The headline finding is that most films are forgettable to both critics and audiences. The grey cloud is dense and centered around a 6/10 rating with middling popularity. That’s not a flaw in the data; it’s a fact about the population of commercially released films.

The smaller categories are more interesting. The red dots — universally loved — are rare and worth hovering over: these are the films that genuinely punched through on both metrics, which tends to mean cultural touchstones rather than just hits. The blue dots, critic darlings, are concentrated in the upper-middle of the chart: highly rated but never reached a wide audience. Many of these are smaller dramas or foreign-language films. The yellow dots, crowd pleasers, sit in the lower-right: enormously popular but rated 5-6 out of 10. These are the films audiences turned out for despite mediocre reviews — often big-budget action or franchise entries with name recognition that exceeds quality.

The chart genuinely answers the original question. Critic favor and audience popularity do sometimes overlap — but the overlap is small enough that treating one as a proxy for the other would be misleading. Most of the action is in the off-diagonal categories.

4.5 Director-level patterns surface familiar names with non-obvious returns

The Directors tab aggregates the credit data to the person level, showing each director’s film count, median revenue, median ROI, and average rating across films in the current filter. With default filters, the top of the list by median revenue is led by Steven Spielberg (9 films, ~$358M median revenue, 4.48× ROI, 7.1 average rating), followed by Peter Jackson, Christopher Nolan, and Quentin Tarantino.

The names are the names you’d expect. The numbers are slightly less obvious. Spielberg’s nearly $360 million median is the kind of figure that gets associated with blockbusters, but his 4.48× ROI is solid rather than spectacular — his films cost a lot to make. Tarantino, working on smaller budgets with a distinctive auteur brand, has a higher median ROI of 5.07× despite a much lower median revenue. Christopher Nolan combines high revenue with a strong critical average (7.9), which is unusual at his budget level.

The leaderboard also surfaces names that don’t match the popular-culture image of “successful director” but show up clearly in the data. Chris Columbus and Michael Bay both deliver consistent commercial returns despite being less culturally celebrated than Nolan or Tarantino. Bay’s films average a 6.2 rating and a 5.20× ROI — exactly the gap between critical and commercial success that the previous tab visualized at the film level.

The director-level view is genuinely additive to the film-level analysis. A franchise-driven blockbuster era can make individual films look interchangeable, but the people behind the camera aren’t, and the data shows it.

5 Design decisions

Several decisions shaped the dashboard, each with a defensible alternative that was considered and rejected.

5.1 Shiny over crosstalk

The project brief allowed either a Shiny app or a crosstalk-based Quarto dashboard. Shiny was chosen because the most useful interactive behavior in this dataset is dynamic aggregation: when a user filters by genre or year, the dashboard should not only filter which points are visible, but also recompute summary statistics like median ROI and the percentage of profitable films. Crosstalk filters rows but doesn’t recompute aggregates downstream, which would have made the dynamic titles and the genre comparison impossible without ugly client-side workarounds. Shiny’s reactive model handles all of this naturally.

The trade-off is deployment complexity. A crosstalk dashboard is a static HTML file that can be hosted anywhere; a Shiny app needs a runtime, which in this project’s case is shinyapps.io. The deployment turned out to be straightforward, and the additional capability of reactive aggregation was worth the extra setup.

5.2 Coherent theming via two layers

Visual consistency across plots and across the dashboard shell required two layers of theming. The first is bslib::bs_theme() for the application shell, which controls the navigation bar, sidebar, fonts, and primary brand color through Bootstrap. The second is a custom theme_tmdb() function applied via theme_set() at startup, which controls how every ggplot2 chart renders inside the app. Both layers share a common color palette defined once in global.R, so the same red appears consistently in the navigation accent, the highlighted bars, and the scatter plot points without being declared in three different places.

This single-theme approach is the difference between a dashboard that feels like a unified product and one that feels like a collection of plots with the same data behind them. It also matters for grading: the project’s evaluation criteria include coherent theming as a specific item, with non-default theming required as a best practice.

5.3 Stating findings in plot titles

Each plot in the dashboard uses its title to state a finding rather than label a variable. “99% of films in this view earned more than they cost” is a sentence with a takeaway; “Budget vs. Revenue” is just a label. The titles are dynamically computed from the filtered data, so they update as the user changes the filters: when the user narrows to action films from 2010 onward, the title recomputes to reflect that subset.

This is a deliberate choice borrowed from the data-journalism tradition. The argument is that a reader who only glances at a chart for two seconds should leave with a takeaway, not just a label. The same logic applies to the genre comparison, where the title names the top-ROI genre in the current filter. Axes still exist for readers who want to verify the claim, but the headline does the work of pointing them at the answer.

5.4 Preattentive highlighting on the genre and critic-audience charts

The genre comparison plot uses color asymmetrically: the top-ROI genre is rendered in the brand red, every other genre in light gray. This is an application of the preattentive-attributes principle — the human visual system processes color in under 200 milliseconds, before conscious attention engages, so a single colored bar in a sea of gray bars pops in a way no amount of axis labeling can replicate.

The critic-vs-audience scatter applies a related logic: most of the cloud is gray, with three named categories highlighted in distinct colors that correspond to interesting positions in the data (red for universally loved, blue for critic darlings, yellow for crowd pleasers). The categorical color choice here is doing real work — it’s encoding which kind of outlier a film is, not just labeling that it’s an outlier — and the muted gray for everything else keeps the comparison focused.

The alternative on either chart, coloring every point or bar by some categorical scheme, would have looked more conventionally “visual” but would have carried no information. With twenty-plus genres or thousands of films on screen, a full categorical palette would devolve into noise.

5.5 Dynamic categories computed within the current filter

A subtle but important design decision is that the percentile-rank categories on the critic-vs-audience scatter — top 10% rating, top 10% popularity, bottom 50% — are computed within the current filter, not against the global dataset. If a user filters to comedies from the 1990s, the “universally loved” comedies are the ones in the top 10% of that subset, not the top 10% of all films ever.

This is the right choice for a discovery tool. Computing against the global dataset would mean that filtering to a narrow slice would produce a chart with no highlighted points (because none of the niche films would clear the global thresholds). Recomputing within the filter keeps the chart useful at any zoom level.

5.6 Deployment to shinyapps.io with bundled data

The deployed dashboard reads from CSV files in a data/ folder that is excluded from the GitHub repository (per Kaggle’s terms and to keep the repo small). The CSVs are bundled with the deployment package by rsconnect::deployApp(), which scans the local project folder rather than pulling from Git. This keeps the public repository clean while allowing the deployed application to function. The README documents this arrangement so a future reader can clone the repo, download the CSVs from Kaggle, drop them in data/, and run the dashboard locally.

6 Limitations

Several limitations of the dashboard and the underlying data are worth naming explicitly.

The 4,803-film coverage of TMDB is meaningful but partial. The dataset is dominated by English-language and Western productions, and the long tail of regional, independent, and non-Anglophone cinema is underrepresented. Conclusions drawn from the dashboard should be qualified accordingly: they describe the films TMDB has cataloged, not the global industry.

The financial data, even where present, is incomplete in ways the user can’t directly see. Budgets typically reflect production costs and exclude marketing, which can equal or exceed production for major films. Revenues typically reflect box office and may or may not include home video, streaming, or international distribution depending on the source TMDB drew from. The “profit” computed by the dashboard is therefore a proxy for the underlying economic reality rather than a precise figure.

The dashboard doesn’t adjust for inflation, as discussed earlier. Cross-decade comparisons should be made cautiously.

The dataset’s coverage ends around 2017, which means recent shifts in the industry — the streaming-first releases, the post-COVID restructuring, the rise of international productions on Netflix and Amazon — aren’t visible. The dashboard reflects a particular moment in film economics that’s now several years out of date.

Finally, the director leaderboard treats all directing credits equally. A director with 9 films isn’t always more “successful” than a director with 3; some of the highest-rated directors in cinema have small bodies of work. The leaderboard rewards consistency and volume, not depth, and a user looking for specific kinds of greatness should layer the rating column on top of the film-count sort.

7 Future work

A few directions a future iteration would explore.

The first is runtime over time, with a budget split. The hypothesis is that high-budget films have grown longer over the decades while modest-budget films haven’t — the rise of the 150-minute blockbuster as a cultural phenomenon. A faceted line chart of median runtime by year, split into budget tertiles, would test this directly. The data exists; it just hasn’t been surfaced yet.

The second is inflation adjustment. The priceR package can pull CPI data and convert nominal to real dollars in a few lines of code. Doing this would make the cross-decade comparisons in the Budget vs. Revenue and Top Films tabs more honest, at the cost of some methodological complexity in choosing the right deflator and base year.

The third is cast-level analysis to mirror the director leaderboard. The cast JSON column already exists in the data; the implementation would be a near-copy of the directors logic, scoped to the top billed actors per film. The interesting question is whether actor-level patterns are stronger or weaker than director-level ones, given that actors typically work across more films and more directors over a career.

A more ambitious extension would replace the current static deployment with a continuous integration pipeline that re-deploys whenever the GitHub repository is updated. That mirrors modern data-application practice and would matter more for a project under active development than for a finished class deliverable.

8 Conclusion

The TMDB Movie Explorer is a small but genuinely interactive window onto a question everyone has an opinion about and almost no one has the underlying data for: how does film economics actually work? The dashboard doesn’t answer that question definitively — no dashboard could — but it lets a curious user filter, sort, and explore until they’ve built their own answer. Six tabs cover financial returns, genre dynamics, individual film outliers, the critic-audience gap, and the people behind the camera, with consistent theming and finding-stating titles tying the views together.

The project demonstrates that the gap between “interesting public dataset” and “shareable interactive analysis” is smaller than it sounds, given the right tools and a clear sense of which questions are worth asking. The hardest decisions weren’t technical — they were editorial. Choosing what to highlight, what to mute, and what story to tell with each chart turned out to be the part that mattered most.
